SSN removed from authentication process | User Merging Discontinued
What Changed

SSN no longer required. The authentication hierarchy was updated: Agents should no longer ask for Social Security information. Instead, require the date and amount of the last transaction recorded for the account.

User merging in Zendesk is no longer required as of January 27. The updated procedure requires updating the ticket requester once a client has been authenticated.

Step 1 • Determine if Authentication is Needed
✅

No Authentication Required


Callers and email senders are considered authenticated when they contact us using the phone number or email address registered on file.

No additional verification needed if this matches our records and pulls the account up in DB/Bizee Tech.

❌

Authentication Required


If the email or phone number shown in Zendesk is not registered in DB/Bizee Tech, you must verify the individual's identity.

Additional steps are required if the account was not located.

Step 2 • Locate the Account

Use the following information in order when trying to pull up an account in DB/Bizee Tech.

A
Order number
B
First and last name of the point of contact
C
Company name
D
DB link on Zendesk (if shown)
Step 3 • Authenticate (Fallback Chain)

Work through each option in order. Only move to the next if the customer cannot provide the current one.

A

Phone number / Email on file

Check if the phone number or email on file matches a profile in the client tab on DB.

If unable to verify — continue to B ↓
B

Payment details

Ask for any of the following:

Last 4 digits of primary card Last 4 of temp. Apple Pay card (via dashboard) PayPal email
If unable to verify — continue to C ↓
C

Billing address

Ask for any of the following:

Street + city Billing zip code
If unable to verify — continue to D ↓
D

Last transaction

Ask for the date and amount of the last charge processed for the account (transaction history).

⚠ If still unable to authenticate, direct to Account Recovery (see below).
Account Recovery

If the caller cannot provide any authentication information, request them to reach out from the email or phone number on file so you can grant access and update the account if required.

Keeping Records Updated (formerly Merging)

If a caller or email sender is not using the contact details on file but has been successfully authenticated, offer to update their account information before ending the interaction.


Suggested verbiage
"Thank you for providing this information. Now that I have access to your account, would you like me to update your phone number/email address on file so we can locate your account faster next time?"

✓ If "Yes"

Send an email requesting them to confirm the contact information on file. Once received, update contact details on DB and in Zendesk's user profile.

✕ If "No"

Change the ticket requester in Zendesk using the contact's email address.
